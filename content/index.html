Development

I've been told an interesting thing about the lathe, a solid piece of equipment you use for working wood or metals. It's the only tool that has the ability to make itself. Which means only the operators ability to think through a job will stand in the way. Now that's all very Zen but it encompasses the philosophy I try to keep with computers. As I'm entirely self-taught, once you've got a computer you have that same ability to solve almost any problem. Especially when you realise to make the next tool with the last tool and keep stepping closer. With the added bonus of it not being 3 tonnes of cast iron and a room full of stock.

Linux

There's a lot of polish on Window that's hard to ignore, it makes for a very pleasant desktop experience.

Using Windows since 2000 was the new hot thing. (Rebooting '98 into DOS to play Command & Conquer doesn't count) Now coming to deploy 10 and 2016 much has changed and it's more interesting than ever.

When it comes to servers that polish starts to hinder, managing details the way windows hides behind wizards and the registry can make it frustrating.

With linux and everything being in a text file somewhere it's a lot more accessible to learn about. I get to use docker on my windows 10 laptop to provision development configurations

With linux I stick to Fedora to get a nice mix of recent packages easily available and steady support.

CoreOS is something I'm starting to experiment with as the update module is very attractive coupled with stacking docker on top to separate the host OS from the application. With a view to making development easier
