Zero SAN Servers

Deploying redundant shared services on the cheap without a SAN

This post aims to detail a method used to provide services to users/clients that are not interrupted by something as trivial as server reboots or power outages. Before we go into too much detail this is a sea of product names and industry names used as product names. It's worth always reminding yourself of the gaol we're even after?

Remember The Goal: No Downtime

In this example I'm going to try to explain the solid underpinnings of a basic but functional network that you could go into the server rack and pull the power from a single host yet nobody would notice. I would define a basic network ans something that provides the following services;

  1. DHCP - For connectivity
  2. DNS - For finding connectivity
  3. Active Directory - For multiple user access
  4. Shared Storage - For data that users want to save and share.
